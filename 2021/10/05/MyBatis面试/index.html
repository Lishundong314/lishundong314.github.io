MyBatis面试

MyBatis是什么

  • MyBatis是半ORM框架，能够通关简单的XML或注解完成配置和映射原生信息，将pojo对象（plain old java object）映射成数据库的记录

  • 内部封装JDBC，开发时只需要关注sql语句本身，不需要花费精力加载驱动，创建连接，创建statement。

优点和缺点

优点

  • 内部封装JDBC，减少了大量的冗余代码，程序员只需关注编写sql语句。
  • 基于sql语句编程，将程序代码和sql分离，降低耦合度，方便统一管理sql
  • 很好的与各种数据库兼容

缺点

  • 数据库移植性差
  • SQL语句编写工作量大（MyBatisPlus可以解决大部分基本的sql语句）

MyBatis与和Hibernate的区别

相同

  • 都是持久层框架，都对JDBC进行了封装

不同

  • MyBatis半自动ORM框架，Hibernate是全自动框架，对sql语句进行了封装，不需要手动编写sql
  • 当代码开发量大时 ，MyBatis性能更好

MyBatis编程步骤

创建xml全局配置文件，再将xml文件以流的形式读取，从而创建出SqlSessionFactory,通过sqlSession就可以获取sqlSession，sqlSession就可以获取到Dao接口的代理类，使得接口和Mapper文件进行绑定，从而使用代理对象完成增删改查。

Dao接口的名与Mapper文件中的namespace一致，标签中的id与Dao接口中的方法一致。

sqlSession也是非线程安全的。

MyBatis工作原理

#{}和${}的区别是什么？

1）#{}是预编译处理，$ {}是字符串替换。

2）mybatis在处理#{}时，会将sql中的#{}替换为?号，调用PreparedStatement的set方法来赋值；mybatis在处理 $ { } 时，就是把 ${ } 替换成变量的值。

3）使用 #{} 可以有效的防止SQL注入，提高系统安全性。

懒加载就相当于把关联查询分两次来做。

Mybatis 仅⽀持 association 关联对象和 collection 关联集合对象的延迟加载，

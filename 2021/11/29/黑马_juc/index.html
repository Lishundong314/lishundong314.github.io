黑马程序员juc笔记

概念

进程和线程

进程

  • 进程就是用来加载指令、管理内存、管理 IO 的
  • 进程就可以视为程序的一个实例。（记事本、穿越火线）

线程

  • 一个进程之内可以分为一到多个线程。
  • 一个线程就是一个指令流，将指令流中的一条条指令以一定的顺序交给 CPU 执行

Java 中，线程作为cpu的最小调度单位，进程作为资源分配的最小单位。

二者对比

  • 进程基本上相互独立的，而线程存在于进程内，是进程的一个子集

  • 进程间通信较为复杂

    ​ 同一台计算机的进程通信称为 IPC（Inter-process communication）

    ​ 不同计算机之间的进程通信，需要通过网络，并遵守共同的协议，例如 HTTP

  • 线程通信相对简单，因为它们共享进程内的内存，一个例子是多个线程可以访问同一个共享变量

  • 线程更轻量，线程上下文切换成本一般上要比进程上下文切换低

并行与并发

区别：是否同一时间进行

并发：交替做不同事的能力

并行：同时做不同事的能力

Java线程

创建并允许线程的三种方法

直接使用 Thread

使用 Runnable 配合 Thread

Java 8 以后可以使用 lambda 精简代码

Thread 与 Runnable 的关系

方法1 是把线程和任务合并在了一起，方法2 是把线程和任务分开了

用 Runnable 更容易与线程池等高级 API 配合

用 Runnable 让任务类脱离了 Thread 继承体系，更灵活

FutureTask 配合 Thread

FutureTask 能够接收 Callable 类型的参数，用来处理有返回结果的情况

Linux查看进程和线程的方法

  • ps -fe 查看所有线程
  • ps -fT -p 查看进程下面的线程
  • kill 杀死进程
  • top 动态的查看线程
  • top -H -p 查看进程下面的线程

线程运行的原理

线程的创建就是在虚拟机栈中，为它分配栈内存。每个栈由多个栈帧组成，每一个java方法就对应一个栈帧，每个栈帧就存放了局部变量表、操作数栈、动态连接方法出口等信息。

线程的运行原理：每一个方法被调用直至执行完毕的过程，就对应着一个栈帧在虚拟机栈中从入栈到出栈的过程。
